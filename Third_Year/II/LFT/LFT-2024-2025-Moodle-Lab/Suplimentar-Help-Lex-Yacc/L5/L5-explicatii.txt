SUMAR pag-1-5: YACC-TEORIE.pdf

Token-ii sunt organizati pe baza regulilor gramaticale; cand o regula este potrivita, este invocata actiunea corespunzatoare.

- Actiunile in C returneaza valori; utilizeaza valorile altor actiuni.

- Analizorul lexical citeste fluxul de intrare + recunoaste token-ii si ii comunica analizorului sintactic.
- Simbol Terminal=Token=structura recunoscuta de analizorul lexical (LEX)
- Simbol non-terminal=structura recunoscuta de analizorul sintactic (YACC)

2.3. Actiuni YACC
- LEX poate returna valori pt. token-i.
XXX : YYY ZZZ { printf ("a message\n");}

- YACC: '$' returneaza o valoare: actiunea seteaza pseudo-variabila $$ la acea valoare.
- pseudo-variabile:$1, $2,..., refera valori returnate de componentele din DREAPTA regulii curente, de la stanga la dreapta.
Ex.
A : B C D ;  $2 valoarea returnata de C; $3 pt. D.
expr : '(' expr ')'	{ $$=$2;}
- Implicit, valoarea regulii=valoarea primului element $1.

--pagina 4: Variabile globale similar cu LEX intre %{ si %}
%{ int var1 = 0; %}
- Analizorul YACC foloseste numai identificatori ce incep cu 'yy', utilizatorii trebuie sa foloseasca alte denumiri.
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
2.4. Analiza lexicala (LEX)
-se furnizeaza un analizor lexical (ex. LEX) care citeste fluxul de intrare si comunica token-ii (cu valori, daca se doreste) analizorului sintactic (YACC)
- Analizorul lexical=Functia yylex() ce returneaza intregi: numarul terminalului ("token-number") cu tipul de token citit.
- Daca este o valoare asociata cu token-ul trebuie atribuita variabilei EXTERNE yylval.
- LEX+YACC trebuie sa se "inteleaga" asupra nr.token sa  comunice corect.
- #define din C permite LEX sa returneze simbolic aceste numere.
Ex: - token DIGIT a fost definit in YACC- in LEX scriem:
{ extern int yylval; int c=getchar(); 
switch(c){case '0':case'1' .........
case'9' :yylval=c-'0';
return(DIGIT);}
}
- Returneaza: cod token DIGIT + valoarea numerica.
- Daca codul pt. LEX e in sect."programe" a fisierului YACC, DIGIT va fi definit ca numarul de token asociat cu token-ul DIGIT.
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
2.5. Modul de lucru al analizorului sintactic (YACC)
- Analizorul generat de YACC=automat finit cu stiva.
- Analizorul citeste+memoreaza simbolul de intrare urmator= token "look-ahead"
- Starea curenta=varful stivei. 
- Starilor automatului le sunt atribuite etichete (intregi mici sau "short int").
- Initial masina e in starea 0 (zero), stiva contine starea 0 si nici un token "look-ahead" nu a fost citit.
